Playbook · Brand variants showcase

One source. Four brands.

Every card below renders the identical content — same hero, same stats, same quote, same helpline. The only thing changing is the brand: colors, typography, logo, shadows, radii. Each variant was produced by editing a single file — _brand.yml — and regenerating tokens. This is the point of the white-label system: a regulator, operator, or jurisdiction can fork Playbook and ship something that looks like theirs in an afternoon, without touching a line of HTML.

To see the same template re-skinned live, open /collateral/rg-page.html and use the Try your brand button to toggle between variants. Source files in /variants/.
